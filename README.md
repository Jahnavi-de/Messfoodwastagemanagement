# Hostel Food Wastage Prediction and Optimization System

A full-stack machine learning web application that predicts hostel food wastage, suggests optimized cooking quantities, stores prediction results in a database, and displays analytical graphs on a dashboard.

## Introduction

Food wastage is a major problem in hostel mess and cafeteria systems. In many institutions, food is prepared based on rough estimates instead of actual demand, which leads to overcooking, higher cost, and resource loss.

This project solves that problem by building a smart prediction system that:
- predicts food wastage using a machine learning model
- suggests optimized cooking quantities
- compares original and optimized waste
- stores results in a MySQL database
- visualizes waste trends on a dashboard

The system is divided into three connected parts:
- `mini-project-main` -> frontend built using Next.js
- `nextjsbackend` -> backend built using PHP and MySQL
- `ml-api` -> machine learning API built using Flask and Python

## Objectives

The main objectives of this project are:
- to predict food waste based on hostel meal data
- to reduce overproduction of food
- to recommend cooking quantities for better planning
- to store all predictions for future analysis
- to provide graphical insights through a dashboard

## Project Architecture


mini-project-main   -> Next.js frontend
nextjsbackend       -> PHP backend + MySQL database
ml-api              -> Flask API + machine learning model


# Workflow
User enters food and attendance details in the frontend.
Frontend sends the request to the PHP backend.
PHP backend forwards the request to the Flask ML API.
ML API predicts food wastage and generates recommendations.
Frontend shows prediction results.
Results are saved in MySQL through the backend.
Dashboard fetches graph JSON from the ML API and displays waste analytics.
